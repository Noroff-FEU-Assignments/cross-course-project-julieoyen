Web Security

How would you advise a potential client to secure his/her website from web security attacks? Note: the word count for your answer should not be greater than 250 words

I would advice the potential client to set up the page using HTTPS so the data sent between the user and the server will be encrypted, and therefore making the data harder to reach for outsiders. Make sure to protect personal identifiable information from web security attacks by using two-factor authentication when making sensitive changes or tasks, and by hiding sensitive data on the page, such as credit card numbers (only showing last four digits) and passwords. Also log out users after a certain amount of inactivity is also important when showing pages with sensitive information. Additionally, I would advise the client to have a team that continues to update the software/plugins and find new ways to keep the page safe. This can be done by e.g. scanning for vulnerabilities, testing, and trying to find weaknesses within the page by doing a mock hack (trying to hack their own page). XSS vulnerability is the most prevalent security threat. A good preventive measure against XSS vulnerability is to eliminate/disable any markup that can giveaway possible hints to run the code. In HTML, for example, it can be tags like < script > and < link >. If it’s obvious where the «important» codes are, it will make it easier for hackers to attack. A post request with a user-specific site-generated secret will make it harder to hack because it looks different for different users and therefore the hackers will have to start from scratch every time.
